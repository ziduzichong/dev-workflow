# 阶段1: 规划与设计

## 执行步骤

1. **执行苏格拉底追问（如 Scope Gate 中未完成）**
   - 目标追问 — "这个功能最终要解决什么问题？谁来使用？使用频率多高？"
   - 边界追问 — "什么情况下不应该触发这个功能？最大数据量/并发量是多少？"
   - 约束追问 — "有哪些技术限制（内存/Flash/延迟/依赖）？有哪些非技术限制（时间/合规/兼容性）？"
   - 验收追问 — "怎么判断这个功能做完了？验收标准是什么？"
   - 追问结束后，将澄清后的需求**回述给用户确认**，不得假设用户已同意

2. **调用 `superpowers:brainstorming` skill** — 需求分析、方案探索、设计核定
3. **调用 `superpowers:writing-plans` skill** — 实现步骤分解、文件改动映射
   - 输出：`$DESIGN_DIR/YYYY-MM-DD-<topic>-design.md`
   - 输出：实现计划（Plan）

## 检查点



| # | 类型 | 验证内容 | 操作 |
|---|------|---------|------|
| 1 | 🔧 | 设计文档文件存在 | `Glob` 搜索 `$DESIGN_DIR/*-design.md`，找到后 `Read` 前 30 行确认非空 |
| 2 | 👤 | 实现计划已获用户确认 | 向用户展示计划摘要，等待确认回复 |
| 3 | 🔧 | 技术栈和依赖已明确 | `Read` 设计文档，检查是否含「技术栈」或「依赖」章节 |


全部通过 → `git commit` → 更新 telemetry → 进入阶段2
