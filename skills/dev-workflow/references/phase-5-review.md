# 阶段5: 代码审查#

## 输入
阶段3/4 完成的所有代码。

## 执行步骤

1. **调用 `superpowers:requesting-code-review` skill**
   - 审查功能完整性、逻辑正确性、安全性、风格规范
   - 对照阶段2 的问题编号列表，逐一验证已解决

2. **调用 `superpowers:receiving-code-review` skill**
   - 生成修改任务表 → `$REPORT_DIR/code-review-issues.md`
   ```
   ## 修改任务表
   | # | 文件 | 行号 | 问题描述 | 严重程度 | 修改方案 | 状态 |
   |---|------|------|---------|---------|---------|------|
   ```
   严重程度: `严重` / `一般` / `建议`

3. **按任务表逐项修改**
   - 严重优先 → 每修复一项编译验证 → 3 次重试上限
   - 复杂 bug 可回到阶段4 流程处理

## 检查点

| # | 类型 | 验证内容 | 操作 |
|---|------|---------|------|
| 1 | 🔧 | 无未关闭的严重/一般问题 | `Read` `$REPORT_DIR/code-review-issues.md`，逐行确认 |
| 2 | 🔧 | 建议类问题已评估 | 检查每个建议项标注了采纳/不采纳原因 |
| 3 | 🔧 | 修改后编译/测试通过 | `Bash` 执行构建+测试 |
| 4 | 🔧 | 阶段2 问题全部回溯验证 | 对照架构审查报告问题列表，逐个标记 |

**全部通过 → `git commit` → 更新 telemetry → 进入阶段6**
