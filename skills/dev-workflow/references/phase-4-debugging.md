# 阶段4: 系统化调试#

## 适用条件

- 阶段3 中无法在 3 次重试内修复的 bug
- 阶段5（代码审查）中发现的复杂逻辑错误
- 用户主动触发的调试请求

**如果当前无待调试问题，直接跳过本阶段进入阶段5。**

## 执行步骤

调用 `superpowers:systematic-debugging` skill，按以下流程处理每个 bug：

```
① 复现 bug → ② 缩小范围（二分法定位）→ ③ 分析根因
                                              ↓
                                         ④ 设计修复方案
                                              ↓
                                         ⑤ 实施修复
                                              ↓
                                   ⑥ 验证修复 + 回归测试
                                              ↓
                                   ⑦ 记录 bug 修复日志
```

**修复后的验证：**
- 原 bug 的复现步骤不再触发问题
- 相关测试用例全部通过
- 无新增编译 warning

## 检查点

| # | 类型 | 验证内容 | 操作 |
|---|------|---------|------|
| 1 | 🔧 | 所有待调试 bug 已处理 | `Read` 读取阶段3 构建日志中的失败记录，逐一确认 |
| 2 | 🔧 | 修复后全量测试通过 | `Bash` 执行全量测试命令 |

**全部通过（或跳过）→ `git commit` → 更新 telemetry → 进入阶段5**
